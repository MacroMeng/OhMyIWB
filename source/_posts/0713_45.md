众所周知，**Git** 是一个十分好用且流行的版本控制系统。但是不是所有人都喜欢 CLI 命令行操作的，即使你记得住 `git commit -am ""` 可能也想找一个好用的图形化 Git 操作面板。下面就来看些 Git GUI 程序吧。

## IN-IDE Git 工具

这些管理器都是 IDE 所集成的。

### VSCode - 不推荐

VSCode 以及其他 VSCode-Based AI IDE 都带有一个 Git 操作面板，它就藏在侧边栏内。但是这个面板不说难用，只能说能用——基于众多开发者评价。即使搭配上第三方插件 GitLens 这些也一样。再加上火焰图都要单独拉上一个插件来做，VSC 是真挺拉的……

### JetBrains IDE - 美味选择

与 VSC 相反，IDEA-like IDE 的 Git 面板一直都受到广泛的好评。与 VSC 只有一个贫瘠的侧边栏空间不同，IDEA 包含了几个独立的分块，包括统一查看 Commit 历史和各类高级操作的 **Git**和专门用于 Commit Commits（~~什么糟糕的短语~~）的**提交**。它的优秀的历史记录图形化与内置火焰图也是众多亮点之一。总之如果你的工作流基于 JB IDE 的话，那么基本上不需要外部工具，JB IDE 做得已经很完美了。

## 独立 Git 工具

### Rebased - 还是 JB⁈

[Rebased](https://github.com/DetachHead/rebased/) 是基于 JetBrains IDE 平台开发的独立软件，砍掉了代码编辑功能。虽然“拿出来做一个独立的 Git 面板”是 JetBrains YouTrack 排名第三的请求，但是这个客户端还是活了没一会就亖了，所以第三方做出来一个社区版就不奇怪了。如果你想要入手一款好用，符合你之前 JB IDE 使用习惯的 Git 面板，Rebased 值得一试。

### GitHub Desktop

Github 官方出品的客户端 [GitHub Desktop](https://desktop.github.com/) 设计简洁，特点是 GitHub 的深度集成。它并非主要为「本地 Git 操作」设计，这点其实是扣分项，因为 Git 不仅仅是 GitHub 的客户端，还有许多 Git 托管方可供选择。不过除去这点，它其实还蛮行：主要就是简洁易懂，工具栏三个大按钮，侧边栏的更改栏，左下方的 Commit 区域，虽然没什么自定义程度可言，但确实是开箱即用的。菜单栏也藏有高级操作可选，如果你主要任务全在 GitHub，并且不想要过多复杂功能全摆在你脸上，可尝试 GitHub Desktop。


### Sourcetree

Sourcetree 也是众多 Git 工具之一。抛开其他的不谈，它的界面就挺让人一言难尽……典型的工业风软件啊。虽然我没怎么用过，不过 Sourcetree 的功能倒是挺齐全，毕竟是工业风软件（x）。但是它坏就坏在要强等账号，没那么自由。

### TortoiseGit

这款软件严格来说不能叫 Git 客户端，因为它活在右键菜单里。我一直对占用我宝贵的神圣而不可侵犯的右键菜单空间的软件避之不及，那就不说这个软件了（其实据说他功能挺全面的）（绝对不是因为我懒）。不过呢它可以在文件图标上显示一个小图标代表同步状态，也算一个亮点吧。


### Git Extensions

这款软件确实能给你带来最为全面，最为深入的 Git 操作。但是你真的能对一个 Win32 风格的风味软件界面产生好感吗？

### Git 自带 GUI

我想不到用它的理由。
